# 开发文档

## 项目是什么

yh-fish 是一个围绕 **Windows 上《异环》游戏窗口** 的本地小工具：**Python FastAPI 后端 + 前端界面**

---

## Release 打包

### 相关文件


| 文件                              | 功能简述                                                                                                                                                          |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `package.json`                  | 根脚本 `build:release`；字段 `**version`**、`**appDisplayName**` 决定安装包/zip 命名与传给 Inno、PyInstaller 的显示名。                                                              |
| `icon.png`                      | 应用图标源图；构建时转为 `release/app-icon.ico`，供 PyInstaller 与 Inno 向导使用。                                                                                                |
| `yh-fish.spec`                  | PyInstaller 配置：入口 `python/main.py`、onedir、无控制台、可选 exe 图标、`webview` 与 `python/images` 等资源。可经环境变量 `**YH_FISH_EXE_BASENAME**` 指定 exe/目录名（与 `appDisplayName` 一致）。 |
| `yi-huan-fish-installer.iss`    | Inno Setup 6 安装脚本（英文 `Default.isl`）；由 `ISCC` 配合 `**/DMyAppVersion` / `/DMyAppName` / `/DMyAppBuildDir**` 注入版本与路径。                                             |
| `scripts/build-release.ps1`     | **主构建流程**：前端 `pnpm build` → 安装构建依赖 → PNG→ICO → PyInstaller → 拷贝 `frontend/dist` → 有 `ISCC` 则打安装包，否则打便携 zip。                                                   |
| `scripts/build-release.sh`      | 占位：提示在 Windows 上使用 `pnpm run build:release`。                                                                                                                  |
| `scripts/png-to-ico.py`         | 将 `icon.png` 导出为多尺寸 `**.ico`**，供 Windows exe / Inno 使用。                                                                                                       |
| `python/requirements-build.txt` | 在运行依赖基础上增加 **PyInstaller** 等构建用包。                                                                                                                             |
| `python/app_paths.py`           | 冻结运行时在 `_MEIPASS` 下定位 `python/images` 等资源（与开发时 `python/` 目录等价）。                                                                                               |
| `python/main.py`                | 仓库根 / 冻结时 exe 旁路径解析、`frontend/dist` 检测；缺资源时对冻结版弹窗提示。                                                                                                          |


### 操作步骤

1. 仓库根：`python -m venv .venv`，再执行 `.venv\Scripts\pip install -r python\requirements-build.txt`。
2. 根目录放置 `**icon.png`**（会生成 `release/app-icon.ico` 给 exe 用）。
3. 在 `package.json` 中维护 `**version**` 与 `**appDisplayName**`。
4. 执行 `**pnpm run build:release**`（或 `powershell -File scripts/build-release.ps1`）。顺序为：前端 build → `pip` → PNG→ICO → **PyInstaller（无控制台 + 图标）** → 拷贝 `frontend/dist` → **收尾**：
  - **若本机能找到 Inno Setup 的 `ISCC.exe`**（默认路径、`PATH`、或环境变量 `**ISCC**` 指向该 exe）：生成 `**release/{appDisplayName}-{version}-setup.exe**` 安装包。
  - **若未安装 Inno**：不报错，改为生成 `**release/{appDisplayName}-{version}-portable.zip`**，解压后运行其中的 `**{appDisplayName}.exe**` 即可。

可选：安装 [Inno Setup 6](https://jrsoftware.org/isdl.php) 后把安装目录加入 `PATH`，或设置用户环境变量 `**ISCC**` = `...\ISCC.exe` 的完整路径，即可在下次构建时得到 `.exe` 安装包。

**体积**：Inno 安装包用 LZMA2，与便携 zip 体积通常接近；大头仍是 PyInstaller 依赖。

**构建产物目录（默认在 `release/`，已 `.gitignore`）**：`release/app/{appDisplayName}/`（exe、`_internal`、`frontend/dist`）、`release/pyinstaller-work`（可删）、`release/app-icon.ico`。
