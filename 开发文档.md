# YH-Fish 开发文档

## 项目是什么

yh-fish 是一个围绕 **Windows 上《异环》游戏窗口** 的本地小工具：**Python FastAPI 后端 + 前端界面**

---

## 架构与数据流（现状）

```
游戏窗口 (HWND)
    → window_capture：按标题正则找窗口、算标题栏裁切高度
    → native_stream (WGC)：整窗抓帧 JPEG
    → capture_service：裁掉标题栏与边缘、更新最新一帧；后台线程按 FPS 循环
    → server：/api/capture/status、/api/capture/fps、/api/capture/mjpeg
    → 前端：状态轮询 + <img> 拉 MJPEG 预览、滑块调 FPS
```

- **入口**：`python/main.py` 启动 uvicorn（默认 `8848`）与 webview；`--dev` 时打开 Vite 开发地址，否则提供 `frontend/dist` 静态资源。
- **窗口匹配**：`capture_service` 默认标题正则为 `^\s*(异环|NTE)\s*$`，与 `window_capture.find_game_hwnd` 一致。
- **前端**：`App` 仅挂载 `CapturePreviewSection`；`api-client` 在开发环境对 MJPEG 直连 `http://127.0.0.1:8848`，避免 Vite 代理缓冲长连接。

---

## 后端 API（已实现）


| 方法   | 路径                    | 说明                                     |
| ---- | --------------------- | -------------------------------------- |
| GET  | `/api/capture/status` | `ok`, `hwnd`, `width`, `height`, `fps` |
| POST | `/api/capture/fps`    | body: `{ "fps": 1–60 }`                |
| GET  | `/api/capture/mjpeg`  | `multipart/x-mixed-replace` JPEG 流     |


## 产品方向

1. 捕获游戏窗口数据
2. 根据设置好的局部截图识别「当前页面」
3. 按页面执行不同逻辑

## 技术注意事项（承接现有代码）

- 非 Windows 或缺少 WGC 后端时，`CaptureService` 会退化为占位图；页面识别若在 Windows 实机上做，应与 `ok && hwnd` 状态联动。
- MJPEG 为预览路径；若识别对 **稳定原图** 有要求，可增加「取单帧 PNG/JPEG」或「仅 ROI」的 API，避免与流式 JPEG 质量/时序耦合过紧。
