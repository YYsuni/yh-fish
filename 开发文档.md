# YH-Fish 开发文档

## 项目是什么

yh-fish 是一个围绕 **Windows 上《异环》游戏窗口** 的本地小工具：**Python FastAPI 后端 + 前端界面**

---

## 架构与数据流（现状）

```
游戏窗口 (HWND)
    → window_capture：按标题正则找窗口、算标题栏裁切高度
    → native_stream (WGC)：整窗抓帧 JPEG
    → capture_service：裁掉标题栏与边缘、更新最新一帧；后台线程按 FPS 循环
    → page_template_match：根据 pages.json 匹配特征值判定页面，特征值需要锁定区域
    → server：/api/capture/status、/api/capture/fps、/api/capture/mjpeg
    → 前端: canvas + ws 绘制预览、滑块调 FPS
```

- **入口**：`python/main.py` 启动 uvicorn（默认 `8848`）与 webview；`--dev` 时打开 Vite 开发地址，否则提供 `frontend/dist` 静态资源。
- **窗口匹配**：`capture_service` 默认标题正则为 `^\s*(异环|NTE)\s*$`，与 `window_capture.find_game_hwnd` 一致。
- **前端**：`api-client` 在开发环境对 MJPEG 直连 `http://127.0.0.1:8848`，避免 Vite 代理缓冲长连接。

---

## 后端 API（已实现）


| 方法   | 路径                    | 说明                                     |
| ---- | --------------------- | -------------------------------------- |
| GET  | `/api/capture/status` | `ok`, `hwnd`, `width`, `height`, `fps` |
| POST | `/api/capture/fps`    | body: `{ "fps": 1–60 }`                |
| GET  | `/api/capture/mjpeg`  | `multipart/x-mixed-replace` JPEG 流     |


## 产品方向

1. 捕获游戏窗口数据
2. 根据设置好的局部截图识别「当前页面」
3. 按页面执行不同逻辑

---

## 页面配置（`python/images/auto_fish/pages.json`）

- **页面**：`pages[]` 每项含 `id`（kebab-case）、`label`（中文说明）、`match_mode`（`any` | `all`）、`features[]`。
- **特征**：`file` 为相对 `pages.json` 所在目录的模板文件名；`region` 为 `[x,y,w,h]`，表示在匹配前对**模板 PNG 本身**先做裁剪，再对整帧做一次 `matchTemplate`。每个特征必须显式配置 `region`（允许越界，运行时会按模板尺寸自动裁剪到有效范围）。返回的 `x,y,w,h` 为匹配到该**裁剪后子图**在识别帧上的像素框（再按预览宽度映射），即子图左上角在画面中的位置。
- **多特征同页**：默认同页 `match_mode: any` 表示任一模板匹配即可判为该页（多条件同时成立可改为 `all`）。
